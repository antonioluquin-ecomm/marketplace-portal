Modelo de Integración

Esta página define cómo se elige el camino técnico para incorporar sellers al Marketplace Sporting. Su objetivo no es solo describir alternativas, sino funcionar como herramienta de decisión para Onboarding, Backlog y planificación en Gantt.

Qué resuelve esta página

Permite identificar qué modelo corresponde para cada seller según su plataforma, madurez tecnológica, capacidad de integración y prioridad comercial.

La regla base es simple: elegir el modelo que permita integrar con menor fricción, mayor control y mejor escalabilidad, evitando desarrollos ad-hoc que no puedan reutilizarse.

Uso operativo

Se utiliza después del relevamiento técnico y antes de pasar un seller al Gantt de ejecución.

  • Backlog: define viabilidad y prioridad.
  • Onboarding: documenta el camino elegido.
  • Gantt: planifica tareas según modelo.

Modelos disponibles

Cada modelo representa un camino distinto de integración, operación y gobierno. Son los 3 únicos caminos soportados.

Rápido

VTEX ↔ VTEX

El seller mantiene su operación en VTEX y Sporting integra la oferta al Marketplace. Ideal para primeras incorporaciones por menor complejidad técnica.

  • Menor time to market.
  • Menor desarrollo inicial.
  • Control parcial desde PIM.
Sin infraestructura propia

Gestión asistida

El seller utiliza herramientas provistas por Sporting para cargar información y operar bajo gobierno de PIM.

  • Mayor control interno.
  • Modelo reutilizable.
  • Base para escalar sellers no VTEX.
API

Sistemas propios

El seller conecta su sistema propio mediante APIs hacia el Seller Center, manteniendo control y trazabilidad desde PIM.

  • Requiere madurez técnica.
  • Escalable a largo plazo.
  • Ideal para sellers grandes.
✅

Reglas de decisión

Criterios mínimos para evitar decisiones ambiguas y sostener el orden operativo.

1
Si el seller usa VTEXPriorizar VTEX ↔ VTEX, salvo que el alcance requiera gobierno completo desde PIM.
2
Si el seller no tiene integración maduraPriorizar Gestión asistida con carga manual o masiva.
3
Si el seller tiene capacidad APIEvaluar Sistemas propios, siempre que el esfuerzo sea reutilizable.
4
Solo 3 modelos soportadosTodo seller debe encuadrarse en VTEX ↔ VTEX, Gestión asistida o Sistemas propios; no se admiten caminos custom fuera de estos tres.
5
Sin modelo definido no pasa a GanttEl Gantt debe planificar ejecución, no resolver la decisión técnica.
6
El modelo define esfuerzoComplejidad, tiempos y responsables dependen del camino seleccionado.
