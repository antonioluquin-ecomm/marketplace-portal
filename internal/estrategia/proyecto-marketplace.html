Proyecto Marketplace
Sporting

Nueva línea de negocio del canal eCommerce: modelo institucional para incorporar sellers externos con criterios claros de integración, operación, rentabilidad y gobernanza. El objetivo es construir una plataforma escalable — no sumar sellers de forma aislada.

Estado actual

Etapa de Activación (Q2 2026). El modelo está definido. El foco es validar la operación real con los primeros sellers, cerrar el marco económico-contractual del piloto y avanzar en el Seller Center en paralelo.

La operación diaria se coordina desde el Portal institucional, el Backlog y el Gantt Operativo. Esta página es la fuente ejecutiva del proyecto.

03Roadmap 2026
Evolución estratégica en tres etapas.
Ver Gantt ↗
Q2
Etapa 1 · Activación

Validar, no escalar

Primeras integraciones con el menor esfuerzo técnico posible. VTEX↔VTEX y Manual/Interfaz.

  • Sellers con VTEX propio
  • Manual / Interfaz
  • QA end-to-end
Q3
Fase 2 · Construcción

Completar la base

El Marketplace opera. Se construye la capa API del Seller Center sin frenar la operación.

  • API interna del SC
  • Planillas operativas
  • Testing por módulo
Q4
Etapa 3 · Escala

Crecimiento autónomo

Seller Center maduro. Sellers con sistemas propios vía API. Criterio de incorporación: conveniencia estratégica.

  • Go-live API por dominio
  • Primer seller vía API
  • Modelo replicable
Criterio de priorización: En la etapa inicial, la capacidad técnica del seller define la prioridad. A medida que el modelo madura, el criterio evoluciona hacia rentabilidad y conveniencia estratégica.
